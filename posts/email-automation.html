Sending personalized emails straight from a spreadsheet

If your team's monthly routine involves opening fifty emails, one per person, copy-pasting a name and a number into each one — that's an afternoon you can get back with about sixty lines of Python.

The manual version, and why it doesn't scale

Mail merge add-ins exist, but most require a specific email client version, break silently on attachments, and offer no visibility when a send fails partway through fifty recipients. A small script gives you full control over all three: template variables, attachments, and error handling.

The building blocks

Three pieces make this work reliably:

  • A source spreadsheet — one row per recipient, with columns matching your template variables (name, department, score, etc.)
  • A template — plain text or HTML with placeholders like {{name}} that get replaced per row
  • A send loop with logging — so a failure on row 32 doesn't silently stop rows 33 through 50, and you know exactly which ones need a retry

Template variables, worked example

A row in your spreadsheet might look like: name=Jordan, department=Coding Ops, score=96. Your template body:

Hi {{name}},

Your quality score for this cycle was {{score}}% in {{department}}.
Great work — details are attached.

The script reads each row, does a straightforward string replace for every {{variable}}, and sends. No manual copy-paste, and the same template serves every recipient.

Attachments, per recipient

The part that trips people up: attachments usually need to be different per person (an individual report, not the same PDF for everyone). The reliable pattern is naming files predictably — e.g. report_{{employee_id}}.pdf — so the script can look up the matching file for each row rather than you attaching them by hand.

Error handling that actually helps

Wrap each send in a try/except, and log successes and failures to a separate results column in the same spreadsheet — not just a console printout that disappears when the terminal closes. That way, re-running the script only needs to process rows marked "failed," instead of resending to everyone.

Note: Variable names and figures here are illustrative. The pattern — template, per-row lookup, logged send — is what transfers to your own data.

What this replaces

For a fifty-person monthly send, this typically turns a 45-minute manual task into a two-minute script run — with a log showing exactly what went out and to whom, which is more auditable than the manual version ever was.
